# WebSocket — Схема базы данных

Реалтайм-система использует три таблицы. `live_sessions` фиксирует жизненный цикл каждой активной сессии. `session_stream_samples` хранит телеметрические данные, накопленные за время сессии. `user_stats` содержит агрегированную долгосрочную статистику пользователя.

## live_sessions

Основная таблица системы. Каждая запись соответствует одному запуску активности пользователем — от `activity:start` до завершения любым из возможных путей.

| Колонка | Тип | Описание |
|---------|-----|----------|
| `id` | uuid PK | |
| `userId` | uuid FK → users | Проиндексирован. |
| `activityType` | varchar | Тип активности, например `breathing` или `generic`. |
| `activityRefType` | varchar nullable | Тип связанной сущности, например `breath_session`. |
| `activityRefId` | uuid nullable | Идентификатор связанной сущности. FK-ограничение отсутствует намеренно — это позволяет расширять набор типов без миграций. |
| `status` | enum | `active`, `disconnected`, `completed`, `abandoned`. |
| `startedAt` | timestamptz | Время начала активности. |
| `disconnectedAt` | timestamptz nullable | Время разрыва соединения. Используется как точка отсчёта grace-таймера. |
| `endedAt` | timestamptz nullable | Время фактического завершения. Выставляется только при чистом `activity:end` или по истечении grace-периода. **Никогда не выставляется в момент разрыва соединения.** |
| `lastActivityAt` | timestamptz | Обновляется при каждом `activity:start` и каждом батче `data:stream`. |
| `metadata` | jsonb | Расширяемое поле для дополнительных данных активности. |
| `createdAt` | timestamptz | |

Индексы: составной по `(userId, status)` — используется для поиска resumable-сессии при переподключении; составной по `(userId, createdAt DESC)` — для истории сессий.

## session_stream_samples

Хранит телеметрию, переданную через `/telemetry`. `StreamEngine` сбрасывает буферы в эту таблицу каждые 5 секунд или при завершении сессии — не на каждый входящий батч.

| Колонка | Тип | Описание |
|---------|-----|----------|
| `id` | uuid PK | |
| `liveSessionId` | uuid FK → live_sessions | Проиндексирован. |
| `dataType` | varchar | Тип данных, например `hrv`, `breath_rate`. |
| `recordedAt` | timestamptz | Клиентская метка времени первого сэмпла в батче. |
| `samples` | jsonb | Массив сырых объектов сэмплов. Схема не фиксирована — зависит от `dataType`. |
| `createdAt` | timestamptz | Время записи на сервере. |

## user_stats

Одна строка на пользователя. Обновляется `StatsService` при каждом завершении квалифицирующей сессии — то есть сессии с продолжительностью не менее `WS_MIN_SESSION_DURATION_S`. Брошенные короткие сессии в статистику не включаются.

| Колонка | Тип | Описание |
|---------|-----|----------|
| `id` | uuid PK | |
| `userId` | uuid FK → users | Unique. |
| `totalSessions` | int default 0 | Количество завершённых квалифицирующих сессий. |
| `totalDurationSeconds` | int default 0 | Суммарное время всех квалифицирующих сессий. |
| `currentStreak` | int default 0 | Текущая серия — количество последовательных дней с хотя бы одной квалифицирующей сессией. |
| `longestStreak` | int default 0 | Рекорд серии за всё время. |
| `lastSessionDate` | date nullable | Дата последней квалифицирующей сессии. Используется для определения разрыва серии. |
| `updatedAt` | timestamptz | |

Серия (`currentStreak`) обнуляется, если между `lastSessionDate` и датой новой сессии — разрыв больше одного календарного дня. Все операции над `user_stats` выполняются через upsert — строка создаётся при первой же завершённой сессии.
