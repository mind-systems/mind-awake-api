# WebSocket — Конфигурация

Все параметры реалтайм-системы задаются через переменные окружения. Значения по умолчанию рассчитаны на типичные условия мобильного использования и не требуют изменений при стандартном деплое.

| Переменная | По умолчанию | Описание |
|------------|-------------|----------|
| `WS_RECONNECT_GRACE_MS` | `30000` | Время ожидания переподключения в миллисекундах. Если пользователь не восстановил соединение за этот период, сессия помечается как `abandoned`. Значение 30 секунд покрывает переключение LTE→WiFi, фоновую оптимизацию ОС и кратковременную потерю сигнала. |
| `WS_MIN_SESSION_DURATION_S` | `10` | Минимальная продолжительность сессии в секундах для учёта в статистике. Сессии короче этого порога не добавляются в `user_stats` — это защита от случайных нажатий и прерванных запусков. |
| `WS_STREAM_MAX_BUFFER_BYTES` | `204800` | Максимальный размер кольцевого буфера телеметрии на одну сессию в байтах (200 КБ). При достижении лимита новые сэмплы вытесняют старые; счётчик вытесненных сэмплов отражается в `data:ack`. |
| `WS_STREAM_MAX_SESSIONS` | `1000` | Максимальное количество сессий с активными буферами телеметрии одновременно. Защищает от исчерпания памяти при пиковой нагрузке. |
| `WS_TELEMETRY_MAX_PAYLOAD_BYTES` | `65536` | Максимальный размер одного сообщения `data:stream` в байтах (64 КБ). Сообщения, превышающие лимит, отклоняются на уровне валидации до попадания в `StreamEngine`. |
| `WS_BACKPRESSURE_SAMPLES_PER_SEC` | `50` | Подсказка обратного давления, которая возвращается клиенту в поле `maxSamplesPerSecond` ответа `data:ack`. Клиент должен самостоятельно снижать частоту отправки до этого значения. |
