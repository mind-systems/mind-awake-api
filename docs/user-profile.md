# User Profile

После аутентификации пользователь может обновить данные своего профиля через `PATCH /user`. Эндпоинт защищён `JwtAuthGuard` — запрос без валидного Bearer-токена получит 401.

## Обновление профиля

Запрос принимает одно или оба поля: `name` (отображаемое имя) и `language` (предпочтительный язык интерфейса). Оба поля опциональны — можно передать только то, что нужно изменить. Допустимые значения `language` ограничены поддерживаемыми локалями (`en`, `ru`); любое другое значение вернёт 400 с описанием ошибки валидации.

`UserService` получает `userId` из JWT-пейлоада через декоратор `@CurrentUser`, затем загружает запись из базы. Если пользователь не найден — возвращается 404. При успешном сохранении возвращается `UserResponseDto` с актуальными данными профиля.

## Язык интерфейса

Поле `language` хранится в таблице `users` и используется для локализации системных писем: при следующей отправке кода сервер возьмёт язык из профиля, если клиент не передаст `locale` явно в `POST /auth/send-code`. Список поддерживаемых локалей и логика нормализации описаны в разделе [Email Auth → Письмо](email-auth.md#письмо).
